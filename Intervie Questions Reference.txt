What is Glide Ajax ?

In ServiceNow, GlideAjax is used to call server-side code (Script Include) from client-side scripts.


👉 GlideAjax = Client → Server communication mechanism

It allows a Client Script to:

Fetch data from server
Execute server logic
Return result without reloading the form

In ServiceNow, if a Catalog Item does not have a Category defined, it typically will NOT be displayed (especially in Service Portal).

🔹 Why does this happen?
👉 1. Service Catalog is Category-driven
The Service Catalog UI (especially Service Portal) displays items based on:
Categories
Catalog → Category → Items hierarchy

👉 If no category is assigned:

The system doesn’t know where to place the item
So it doesn’t render it in the catalog navigation


👉 “If a catalog has no categories, catalog items won’t be visible in Service Portal browsing because the UI and widgets are category-driven and require categories to display items.”


If catalog has no categories
👉 “Yes, the catalog will be visible, but without categories it will appear empty because Service Portal displays items through categories.”
